# Codex Working Preferences

## Layout implementation rule
- 사용자 제공 레이아웃 코드는 임의 스타일을 추가하지 않고 그대로 반영한다.
- 요청하지 않은 테두리(`outline`, `border`), 곡률(`rounded-*`), 색상 오버레이/배경색을 추가하지 않는다.
- 편집 기능 연결이 필요할 때도 시각 스타일은 유지하고, 이벤트 바인딩만 최소한으로 추가한다.
- 사용자 제공 경로/폴더를 절대 임의 변경하지 않는다. 지정 경로가 있으면 그 경로만 사용한다.
- 사용자 제공 파일명 규칙을 절대 임의 변경하지 않는다.
- 사용자 요청이 없는 한 디자인 개선/정리/리팩터링을 하지 않는다.
- 사용자 요청 변경사항은 적용 후 반드시 2회 확인한다.
- 1차 확인: 코드 값/경로가 정확히 반영되었는지 파일에서 직접 확인한다.
- 2차 확인: 화면 미반영 가능성(기존 저장 데이터/캐시/레이아웃별 키 차이)을 점검해 원인을 함께 안내한다.
